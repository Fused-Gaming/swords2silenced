# MCP Integration Guide

This guide explains how to integrate the Fused-Gaming MCP skills with the swords2silenced monorepo structure.

## Architecture Overview

```
swords2silenced/
├── apps/
│   ├── web/                    # Frontend application
│   │   └── src/
│   │       ├── components/     (generated by frontend-design)
│   │       ├── themes/         (generated by theme-factory)
│   │       ├── art/            (generated by algorithmic-art)
│   │       └── svg/            (generated by canvas-design)
│   └── ...
├── packages/
│   ├── ui/                     # Shared UI package
│   ├── api/                    # API package
│   └── core/                   # Core business logic
├── tools/
│   ├── mcp/                    # MCP skills (YOU ARE HERE)
│   │   ├── src/
│   │   ├── config.json         # Skill configurations
│   │   ├── .env                # Environment variables
│   │   └── README.md           # Main documentation
│   ├── cli/
│   └── generators/
├── package.json                # Root monorepo config
├── tsconfig.json               # Root TypeScript config
└── .github/
    └── workflows/              # CI/CD pipelines
```

## Step 3: Configure for Each App

### For apps/web

Update `apps/web/package.json`:
```json
{
  "scripts": {
    "generate:components": "npx skill frontend-design generate",
    "generate:theme": "npx skill theme-factory generate",
    "validate": "npm run validate --workspace=@swords2silenced/mcp-skills"
  }
}
```

### For packages/ui

```bash
# Generate shared components
npx skill frontend-design generate Button Badge Input
npm run build --workspace=packages/ui
```

### For packages/core

```bash
# No direct MCP integration needed
# Uses generated components from packages/ui
```

## Step 4: TypeScript Path Configuration

Verify `tsconfig.json` includes MCP paths:

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["packages/*"],
      "@apps/*": ["apps/*"],
      "@tools/*": ["tools/*"],
      "@mcp/*": ["tools/mcp/*"]
    }
  }
}
```

Update workspace `tsconfig.json` files:

```json
{
  "extends": "../../tsconfig.json",
  "compilerOptions": {
    "outDir": "./dist",
    "rootDir": "./src"
  }
}
```

## Step 5: Build Integration

### Root package.json Scripts

Add or verify build scripts:
```json
{
  "scripts": {
    "build": "npm run build --workspaces",
    "validate": "npm run validate --workspace=@swords2silenced/mcp-skills",
    "generate": "npm run generate:theme && npm run generate:components"
  }
}
```

### Build Order

1. Build MCP skills first
2. Generate themes and components
3. Build UI package
4. Build frontend app
5. Build API package

```bash
# Recommended build script
npm run build --workspace=@swords2silenced/mcp-skills && \
npm run build --workspace=packages/ui && \
npm run build --workspace=apps/web && \
npm run build --workspace=packages/api
```

## Step 6: CI/CD Integration

### GitHub Actions

Create `.github/workflows/mcp-build.yml`:

```yaml
name: MCP Build & Validation

on:
  push:
    branches: [main, develop]
    paths:
      - 'tools/mcp/**'
      - 'apps/web/**'
      - 'packages/ui/**'
  pull_request:
    branches: [main]

jobs:
  mcp:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - uses: actions/setup-node@v3
        with:
          node-version: '18'
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Build MCP skills
        run: npm run build --workspace=@swords2silenced/mcp-skills
      
      - name: Run MCP tests
        run: npm run test --workspace=@swords2silenced/mcp-skills
      
      - name: Validate deployment
        run: npm run validate
```

### Validation Before Deployment

Add to deployment workflow:
```yaml
- name: Pre-deploy validation
  run: npm run validate -- --fail-on-warning
  
- name: Check component generation
  run: npx skill frontend-design list
```

## Step 7: Component Generation Workflow

### Generate Components for Web App

```bash
# 1. Generate individual component
npx skill frontend-design generate Button --name=PrimaryButton

# 2. This creates:
#    apps/web/src/components/PrimaryButton/
#    ├── PrimaryButton.tsx
#    ├── PrimaryButton.module.css
#    ├── PrimaryButton.test.tsx
#    ├── PrimaryButton.stories.tsx
#    └── index.ts

# 3. Export from shared package
# packages/ui/src/index.ts
export { default as PrimaryButton } from '@apps/web/components/PrimaryButton';

# 4. Use in other packages
// packages/core/src/dashboard.tsx
import { PrimaryButton } from '@swords2silenced/ui';
```

### Generate Themes

```bash
# 1. Generate default theme
npx skill theme-factory generate --primary="#4F46E5"

# 2. Creates:
#    apps/web/src/themes/
#    ├── default/
#    │   ├── colors.ts
#    │   ├── typography.ts
#    │   ├── spacing.ts
#    │   └── index.ts
#    └── dark/
#        └── ... (dark variant)

# 3. Apply theme in app
// apps/web/src/main.tsx
import { createTheme } from '@apps/web/themes/default';
```


## Common Integration Patterns

### Pattern 1: Component Generation Pipeline

```
User Request
    ↓
ASCII Mockup (wireframe)
    ↓
Frontend Design (generate component)
    ↓
Tests Added
    ↓
Export to UI Package
    ↓
Use in Applications
```

### Pattern 2: Theme and Component Sync

```
Theme Factory (generate colors, typography)
    ↓
Frontend Design (use theme colors in components)
    ↓
All Components Styled Consistently
    ↓
Easy Theme Switching (dark mode, etc.)
```

### Pattern 3: Custom Tool Development

```
ASCII Mockup (design tool UI)
    ↓
Skill Creator (create tool logic)
    ↓
MCP Builder (create backend service)
    ↓
Frontend Design (create tool components)
    ↓
Integrated Tool Ready
```

## Monorepo-Specific Configuration

### Per-Workspace Configuration

Create `apps/web/mcp.json`:
```json
{
  "skills": {
    "frontend-design": {
      "outputDir": "./src/components",
      "templates": "../../tools/mcp/templates/components"
    },
    "theme-factory": {
      "outputDir": "./src/themes",
      "baseColor": "#4F46E5"
    }
  }
}
```

### Shared Configuration

Update `tools/mcp/config.json`:
```json
{
  "monorepo": {
    "rootDir": "../../",
    "workspaces": [
      "apps/web",
      "packages/ui"
    ]
  }
}
```

## Performance Considerations

### Lazy Load MCP Skills

Only enable needed skills in config:
```json
{
  "skills": {
    "frontend-design": { "enabled": true },
    "theme-factory": { "enabled": true },
    "algorithmic-art": { "enabled": false }
  }
}
```

### Cache Generated Assets

```bash
# Generated components are cached
# Clear cache if needed
rm -rf apps/web/src/components/.cache
npm run generate
```

## Troubleshooting Integration

### Issue: Component imports fail

```bash
# Verify path aliases in tsconfig.json
npm run type-check

# Verify component export in index.ts
cat packages/ui/src/index.ts
```

### Issue: Theme not applying

```bash
# Check theme generation
npx skill theme-factory list

# Verify theme import
grep -r "from '@apps/web/themes'" apps/web/src/
```

### Issue: MCP skills not found

```bash
# Rebuild MCP skills
npm run build --workspace=@swords2silenced/mcp-skills

# Check installation
npm list @swords2silenced/mcp-skills
```
