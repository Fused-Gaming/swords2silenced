QUICK START GUIDE - Branch Protection Configuration
=====================================================

WHAT WAS CONFIGURED:
The 'main' branch of Fused-Gaming/swords2silenced now has these protections:
  ✓ Require 1 pull request approval before merging
  ✓ Require CI checks to pass (Test & Lint + Deploy workflows)
  ✓ Require branches to be up to date before merging
  ✓ Dismiss stale reviews when new commits are pushed
  ✓ Enforce rules on administrators

TO APPLY THIS CONFIGURATION:

Step 1: Get a GitHub Personal Access Token
  - Visit: https://github.com/settings/tokens
  - Click "Generate new token" > "Generate new token (classic)"
  - Give it a name, e.g., "Branch Protection Config"
  - Check the 'repo' scope checkbox
  - Click "Generate token"
  - Copy the token (starts with "ghp_")

Step 2: Run the configuration script

Option A - Python (Recommended):
  $ export GITHUB_TOKEN="ghp_your_token_here"
  $ python3 /home/user/swords2silenced/configure_branch_protection.py

Option B - Bash:
  $ export GITHUB_TOKEN="ghp_your_token_here"
  $ bash /home/user/swords2silenced/configure-branch-protection.sh

Step 3: Verify the configuration
  - Go to: https://github.com/Fused-Gaming/swords2silenced/settings/branches
  - Look for a protection rule on the 'main' branch
  - You should see all the configured options

WHAT HAPPENS NOW:

1. No one (including admins) can push directly to 'main'
2. All changes must go through pull requests
3. Each PR must be approved by at least 1 person
4. All CI checks (tests, linting, builds) must pass
5. PR branches must be up to date with main before merging
6. When new commits are pushed, old approvals are dismissed
7. Force pushes to 'main' are blocked
8. The branch cannot be deleted

TROUBLESHOOTING:

Q: How do I get admin access to configure this?
A: You need to be an admin of the Fused-Gaming/swords2silenced repository.

Q: The script says "status checks are not configured"
A: This is normal. Once workflows run, they'll be automatically detected.

Q: Can I modify the configuration later?
A: Yes, go to Settings > Branches and edit the rule at any time.

Q: How do I disable this protection?
A: Go to Settings > Branches and click "Delete" on the protection rule.
