# Email Service Setup for Swords to Silenced

## Overview

The email service is configured using **Nodemailer** with SMTP connection. This enables the website to:
- Send contact form submissions to admin
- Send confirmation emails to users
- Send notifications and alerts

## Configuration Files

### `.env.local` (Root Directory)
Contains all sensitive email credentials. This file is:
- ✅ **Gitignored** - Never committed to version control
- 🔐 **Contains secrets** - SMTP password and credentials
- 🔧 **Environment-specific** - Different per deployment (dev, staging, prod)

### Email Service Module
- **Location**: `apps/web/src/lib/emailService.ts`
- **Exports**:
  - `sendEmail(options)` - Send an email
  - `verifyEmailConfig()` - Verify SMTP connection

### API Endpoints

#### POST `/api/contact`
Handles contact form submissions.

**Request Body:**
```json
{
  "name": "User Name",
  "email": "user@example.com",
  "subject": "Contact Subject",
  "message": "Message body"
}
```

**Response:**
```json
{
  "success": true,
  "message": "Message sent successfully. We will be in touch soon."
}
```

#### GET `/api/email-status`
Verifies email configuration is working.

**Response:**
```json
{
  "configured": true,
  "status": "SMTP connection verified",
  "config": {
    "host": "smtp.swordstosilenced.com",
    "port": "587",
    "secure": "false"
  }
}
```

## Setup Instructions

### 1. Configure SMTP Credentials

Edit `.env.local` in the project root:

```bash
# SMTP Server Configuration
MAIL_HOST=smtp.swordstosilenced.com
MAIL_PORT=587
MAIL_USER=contact@swordstosilenced.com
MAIL_PASS=your-app-specific-password
MAIL_FROM=contact@swordstosilenced.com
MAIL_FROM_NAME=Swords to Silenced
MAIL_SECURE=false

# Admin Email Addresses
ADMIN_EMAIL=admin@swordstosilenced.com
PRESS_EMAIL=press@swordstosilenced.com
SUBMISSIONS_EMAIL=submissions@swordstosilenced.com
```

### 2. Get SMTP Credentials

For `swordstosilenced.com` domain, you'll need:

**Option A: Using Domain Email Provider**
- If you use cPanel/Plesk: Use the email account credentials
- If you use Google Workspace: Generate an app password
- If you use Microsoft 365: Generate an app password
- If you use custom mail server: Use the server's SMTP details

**Option B: Using Third-Party Email Service**
Popular options:
- **SendGrid** - `smtp.sendgrid.net:587` (username: `apikey`)
- **AWS SES** - `email-smtp.region.amazonaws.com:587`
- **Mailgun** - `smtp.mailgun.org:587`
- **Postmark** - `smtp.postmarkapp.com:587`
- **Brevo** - `smtp-relay.brevo.com:587`

### 3. Install Dependencies

Dependencies are already installed:
```bash
npm install nodemailer @types/nodemailer --save
```

### 4. Test Email Configuration

Create a test script or use the status endpoint:

```bash
curl http://localhost:3000/api/email-status
```

Expected success response:
```json
{
  "configured": true,
  "status": "SMTP connection verified",
  "message": "Email service is configured and ready"
}
```

## Email Flow

### Contact Form Submission

1. **User submits form** → `/api/contact` endpoint
2. **Validation** → Check required fields and email format
3. **Admin email** → Sends to `ADMIN_EMAIL` with:
   - User's name, email, subject, message
   - Reply-To set to user's email
   - Formatted HTML body
4. **Confirmation email** → Sends to user's email with:
   - Acknowledgment message
   - Copy of their message
   - Links to website
5. **Response** → Returns success/error status to frontend
6. **UI feedback** → Shows success or error message

## Security Considerations

### ✅ Best Practices

1. **Never commit credentials**
   - `.env.local` is in `.gitignore`
   - Use environment-specific configurations

2. **Use app-specific passwords**
   - Gmail, Microsoft 365, etc. support app passwords
   - More secure than primary password
   - Can be revoked independently

3. **Enable TLS encryption**
   - Port 587 with TLS (recommended)
   - Port 465 with SSL (legacy but still secure)
   - Never send credentials over unencrypted connection

4. **Validate input**
   - Email format validation
   - Message length limits
   - Rate limiting (if needed)

5. **Error handling**
   - Don't expose SMTP errors to users
   - Log errors server-side
   - Provide generic error messages

6. **Monitor deliverability**
   - Check spam folder
   - Monitor bounce rates
   - Verify DNS records (SPF, DKIM, DMARC)

## DNS Configuration for Email Delivery

For best deliverability, configure:

### SPF Record
```
v=spf1 include:_spf.google.com include:sendgrid.net -all
```

### DKIM Record
Generated by your email provider

### DMARC Record
```
v=DMARC1; p=quarantine; rua=mailto:admin@swordstosilenced.com
```

## Troubleshooting

### Email Not Sending

1. **Check configuration**
   ```bash
   curl http://localhost:3000/api/email-status
   ```

2. **Verify credentials**
   - SMTP host correct?
   - Port correct? (587 for TLS, 465 for SSL)
   - Username/password correct?
   - App-specific password (not primary)?

3. **Check firewall**
   - Is SMTP port open?
   - Does ISP block outgoing mail?
   - Is VPN/proxy interfering?

4. **Check logs**
   - Server logs for errors
   - Email provider's bounce notifications
   - Spam folder

### Emails Going to Spam

1. **Improve email authentication**
   - Configure SPF, DKIM, DMARC
   - Use consistent From address
   - Include unsubscribe link

2. **Quality content**
   - Avoid spam trigger words
   - Don't use misleading subject lines
   - Include company information
   - Professional formatting

3. **Reputation**
   - Use dedicated IP (if available)
   - Monitor bounce rates
   - Remove invalid addresses
